以下の仕様で、自身のホームページの実際のページ遷移用ローディング演出を実装してください。

【重要】
- 遷移用の専用HTMLページを間に挟まないこと
- ページ遷移は「遷移元ページ → 遷移先ページ」の1回だけにすること
- 遷移元ページで前半演出を行い、遷移先ページで後半演出を再開すること
- 現状の問題は「遷移後に元の遷移先ページが一瞬出てから、さらに演出後のページに移る」ことなので、それを起こさない実装にすること
- 遷移先ページ読み込み直後に、素のページが一瞬でも見えないようにすること
- そのために、必要なら遷移先ページの head の早い段階で sessionStorage を読み、html にクラスを付与して初回描画前から表示状態を制御すること

【目的の見え方】
- サイト間の移動時にローディング画面を出す
- 背景演出は以下
  - 0と1の数字が、上下から中央へ向かって流れる
  - 16進数コードが、左右から中央へ向かって流れる
  - ニューロン風の発光やネットワークを感じるイメージ
- 将来的に、奥から手前へ小走りに来るアニメキャラクター風の女性を追加しやすいよう、キャラクター用レイヤーを後から足せる構造にすること
- 今回はキャラクター自体は実装しなくてよい

【ローディングバー】
- 演出的な進行でよい
- 表示は 0% → 16% → 44% → 72% → 99%
- 演出時間は約3秒

【表示ルール】
- 0%〜16%
  - 遷移元ページが薄くなっていく
- 16%〜72%
  - ページ画像は表示しない
  - オーバーレイ演出のみを見せる
- 72%〜99%
  - 遷移先ページが徐々に濃くなっていく
- 完了時にオーバーレイを消して遷移先ページを完全表示する

【実装ルール】
- 実際の遷移は1回だけ
- 遷移元で sessionStorage に開始時刻、遷移状態、遷移先URLなどを保存すること
- 遷移元では 16% まで遷移元ページを薄く見せる
- 遷移元ページが見えなくなったあと、16%〜72% のどこか適切なタイミングで直接遷移先ページへ移動すること
- 遷移先ページでは sessionStorage を読んで後半演出を再開すること
- 遷移先ページでは 72% までは本体を見せないこと
- 遷移先ページでは 72% 以降に本体をフェードインまたは徐々に濃くすること
- 遷移先ページが読み込まれた瞬間に素のページが見えないようにすること
- 外部ライブラリは使わない
- HTML / CSS / JavaScript だけで実装する
- 共通CSS / 共通JS に分離して再利用しやすくすること

【背景演出の実装方針】
- 0と1は canvas で、上下から中央に向かうように動かす
- 16進数は DOM または CSS アニメーションで、左右から中央に向かう複数行の流れにする
- ニューロン風の点や線、発光表現を軽量に追加する
- 画面中央へ収束する感じが出るようにする

【UI】
- オーバーレイ中央または下中央にローディングパネルを配置
- タイトル、状態メッセージ、パーセンテージ、ローディングバーを表示
- サイバー感、発光感のある見た目にする

【出力方針】
- どのファイルを追加・編集するか明示すること
- 遷移元ページ、遷移先ページ、共通CSS、共通JS の実装をまとめて出すこと
- 出力は説明だけでなく実装コードも含めること
